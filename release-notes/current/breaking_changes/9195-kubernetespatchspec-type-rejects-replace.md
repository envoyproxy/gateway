The `type` field of Kubernetes resource patches (`KubernetesPatchSpec`) now only accepts `StrategicMerge` or `JSONMerge`; any other value, including `Replace`, is rejected. For the EnvoyProxy CRD this is enforced at admission via the CRD schema (an OpenAPI `enum`); at reconciliation time it is rejected for every patch path, including the Envoy proxy Deployment, Service, DaemonSet, and HPA and the rate limit Deployment and HPA (the latter configured via non-CRD EnvoyGateway configuration, which CRD admission does not cover). Previously `Replace` was accepted but replaced the entire generated resource with the patch value instead of merging into it.
