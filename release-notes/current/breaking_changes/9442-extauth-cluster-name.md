The Envoy cluster generated for a `SecurityPolicy` external authorization service is now named `extauth/<authority>/<hash>` (derived from the backend and its settings) instead of `securitypolicy/<namespace>/<name>/extauth/0`. Cluster stat names change accordingly (for example, Prometheus/Datadog `cluster.securitypolicy_<ns>_<name>_extauth_0.*` becomes `cluster.extauth_<authority>_<hash>.*`), so dashboards or alerts scraping the old names, and any EnvoyPatchPolicy or extension-server logic matching the old cluster name, must be updated. The ext_authz HTTP filter name is unchanged.
