Envoy Gateway now reconciles TCPRoute and UDPRoute using the `gateway.networking.k8s.io/v1` API group, promoted in Gateway API v1.6. To avoid traffic disruption, upgrade the Gateway API CRDs to v1.6 **before** upgrading Envoy Gateway. Users of the **standard channel** must also update their TCPRoute and UDPRoute manifests to `apiVersion: gateway.networking.k8s.io/v1` before upgrading the CRDs, since `v1alpha2` is no longer served in the v1.6 standard channel. Users of the **experimental channel** can upgrade the CRDs first; both `v1` and `v1alpha2` continue to be served, so existing manifests work without immediate changes. If the v1.6 CRDs are not installed before upgrading Envoy Gateway, TCP and UDP routes will be silently skipped by the reconciler. The stored version in etcd moves from `v1alpha2` to `v1`; a storage-version migration will be required before `v1alpha2` is eventually removed.
