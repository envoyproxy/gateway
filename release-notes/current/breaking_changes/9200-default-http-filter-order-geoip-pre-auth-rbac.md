The default generated HTTP filter chain order has changed: the GeoIP filter and a new internal pre-auth RBAC filter now run before the authentication filters (`ext_authz`, OAuth2, basic auth, API key auth, JWT). This lets authorization rules that are independent of authentication state (matching only on `clientCIDRs` and/or `clientIPGeoLocations`) be enforced before authentication. If you rely on an `EnvoyPatchPolicy` or an Extension Server that targets the GeoIP, authentication, or RBAC filters by position, review and update those patches.
