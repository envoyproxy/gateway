Fixed an issue where an HTTPRoute rule whose filters configure a `DirectResponse` or `Redirect` (e.g. `RequestRedirect`) could be overwritten with a `500` `DirectResponse` when another filter in the same rule fails to process. The route now preserves the filter-configured `Redirect`/`DirectResponse`, matching the existing `GRPCRoute` behavior. The filter error is still surfaced in the HTTPRoute `Accepted` status condition.
